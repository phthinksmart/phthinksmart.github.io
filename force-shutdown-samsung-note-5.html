PEMBAHASAN :

⇒ Menyalakan Komputer sesuai dengan Perintah

Prosedur

Prosedur artinya sesuai dengan langkah dan tahapan - tahapan yang betul. Kekuatan dan performa komputer dipengaruhi oleh berbagai hal, seperti menyalakan dan mematikan komputer yang dipakai.

Proses menyalakan komputer disebut booting, yang terdiri dari cold booting, dan warm booting.

Hal - Hal yang harus Diperhatikan sebelum Melakukan Booting :

Memastikan perangkat - perangkat pada komputer sudah terpasang dengan betul.Memastikan aliaran listrik sudah mengalir ke dalam stabilizer yang udah terhubung ke CPU dan juga monitor.Memastikan gak ada perangkat - perangkat tambahan pada komputer yang dipasang.Cold Boot

Cold atinya dingin, jadi kalau cold boot mksudnya adlah menjalankan komputer dmna komputer dalam kondisi masih dingin atau dalam keadaan ga hidup.

Langkah - Langkah Melakukan Cold Booting :

Komputer dalam keadaan gak hidup.Pencet tombol Power pada CPU.Pencet tombol Power pada monitor.Tunggu sampai proses booting selesai hinga muncul tampilan dekstop pada layar monitor.Warm Boot

Warm boot artinya menjalankan komputer sedang komputer dalm keadaan menyala. Warm boot dilakukan pada saat ada masalah pada komputer, misal hank, error yang disebabkan oleh system crash, terinfeksi virus atau masalah lain. Gejala komputer yang bermasaalah diantarnya gak bisa bekerja mouse dan keyboard pada komputer.

Langkah - Langkah Melakukan Warm Boot :

Komputer berkeadaan idup.Pencet tombol reset di CPU.Komputer akan mati sebentar kemudian idup lagi dan melakukan booting.Setelah booting selesai entar nanti komputer biasanya melakukan scandisk karna mematikan komputer dengan dipaksa.Nanti akan tampil scandisk , tungu sampe proses scan complete 100%.Komputer nanti akan normal kembali.

⇒ Mematikan Komputer (Turn Off/Shut Down) sesuai dengan Perintah

Shut Down dari Dekstop

Shut down harus dilakukan dengan memakai mouse.

Langkah - Langkah Melakukan Shut Down dari Dekstop :

Pastiin semua program atau apk udah tertutup.Klik Start di taskbar.Klik Shut down.Tungu sampe CPU mati sendiri.Kalau CPU udah mati, matiin monitornya dengan cara memencet tombol power di monitor.Shut Down dari Keyboard

Pastiin semua program atau apk uduh tertutup.Pencet tombol Alt + F4 pada keyboard.Nanti akan muncul jendela Shut Down Windows pada dekstop.Pilih 'Shut down' trus pencet OK.Tungu hinga CPU mati sendiri.Matiin monitor dengan memencet tombol power pada monitor.Pelajari Lebih Lanjut :Materi mengenai reboot dapat di cek pada link berikut → https://brainly.co.id/tugas/11926453.Materi mengenai pengertian warm boot dapat di cek pada link berikut → https://brainly.co.id/tugas/11181939.Materi mengenai yang dimaksud proses menyalakan komputer dengan menggunakan tombol reset dapat di cek pada link berikut → https://brainly.co.id/tugas/14822855.

9. jelaskan maksut dari perintah command prompt berikut ini - shutdown. -s. - shutdown - R​


Perintah Shutdown Ubuntu dan CentOS

Perintah ‘shutdown’ Ubuntu dan CentOS ini berfungsi untuk mematikan, reboot, dan menghentikan sistem perangkat. Di bawah ini adalah perintah syntax untuk shutdown dan restart.

Shutdown [OPTIONS] [TIME] [WALL]

Argumen pertama dari [OPTIONS] adalah string waktu. Setelah memilih argumen ini, selanjutnya Anda dapat mengetikkan wall message untuk memperingati user yang sudah login bahwa sistem akan dimatikan.

Format untuk string waktunya adalah hh:mm (jam/menit), sedangkan format waktunya sendiri menggunakan aturan 24 jam. Fungsi dari string ini untuk menunjukkan waktu kapan perintah shutdown akan dijalankan. Anda juga bisa menggunakan elemen +m yang mengindikasikan menit. Instruksi ini akan memerintahkan perangkat untuk mati secara otomatis dalam hitungan menit yang dikehendaki.

Untuk mematikan VPS saat itu juga, Anda bisa menggunakan syntax now yang adalah alias untuk +0. Jika Anda tidak menentukan argumen [time], maka Linux akan mengaturnya ke +1 secara default sebelum shutdown dilakukan.

Perlu diingat bahwa Anda harus menentukan argumen waktu jika hendak membuat wall message. File /run/nonlogin akan dibuat 5 menit sebelum sistem dinonaktifkan asalkan Anda telah menetapkan argumen waktu untuk memperingati user yang hendak login.
